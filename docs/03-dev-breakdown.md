# 微信小游戏开发拆解：极简横版武器进化爬塔 Roguelite

## 一、开发目标
目标不是一次做大，而是先做出一个能验证方向的 MVP。

MVP 只验证三件事：
1. 武器进化爽不爽
2. 爬塔是否让人想继续打
3. 广告变现是否能自然嵌入

---

- 核心战斗循环

- 三选一强化系统

## 三、系统拆解

## 3.1 战斗主循环
### 功能目标
搭建最基础的“进入战斗 → 清怪 → 升级 → 选强化 → 继续战斗 → 失败结算”闭环。

### 开发任务
- [ ] 场景初始化
- [ ] 玩家角色生成
- [ ] 怪物刷出逻辑
- [ ] 自动攻击触发逻辑
- [ ] 命中 / 扣血 / 死亡逻辑
- [ ] 经验掉落或经验累积
- [ ] 升级弹窗
- [ ] Game Over 结算
- [ ] 一键再来一局

### 验收标准
- 用户进入 10 秒内开始打怪
- 60 秒内至少升级 2 次
- 死亡后 2 秒内可重新开始

---

## 3.2 玩家系统
### 功能目标
提供一个轻量但稳定的玩家战斗实体。

### 基础属性
- [ ] HP
- [ ] Attack
- [ ] AttackSpeed
- [ ] CritRate
- [ ] CritDamage
- [ ] MoveSpeed

### 行为
- [ ] 左右移动
- [ ] 跳跃（可选）
- [ ] 冲刺（可选）
- [ ] 自动攻击

### 验收标准
- 玩家操作不复杂
- 自动攻击正常工作
- 不依赖高操作技巧

---

## 3.3 武器系统
### 功能目标
让武器成为这款游戏最核心的爽点来源。

### 首版建议武器
- [ ] 回旋剑
- [ ] 分裂弓
- [ ] 火焰杖
- [ ] 雷暴锤

### 每把武器要实现的内容
- [ ] 基础攻击形态
- [ ] 升级数值变化
- [ ] 至少 1 次明显进化
- [ ] 命中特效
- [ ] 音效差异

### 验收标准
- 每把武器的手感明显不同
- 至少有一把武器进化前后差异巨大
- 玩家能明显感知“这把武器变强了”

---

## 3.4 强化与三选一系统
### 功能目标
让用户持续获得“变强选择”，形成轻构筑体验。

### 任务
- [ ] 升级时暂停战斗
- [ ] 弹出三选一 UI
- [ ] 强化池随机抽取
- [ ] 选择后即时生效
- [ ] 避免无效项过多重复

### 首版强化类别
- [ ] 攻击+
- [ ] 攻速+
- [ ] 暴击+
- [ ] 穿透+
- [ ] 分裂+
- [ ] 连锁+
- [ ] 吸血+
- [ ] 范围+
- [ ] 指定武器升级

### 验收标准
- 升级频率足够高
- 每个强化都能被用户快速理解
- 选完强化后体感明显变化

---

## 3.5 怪物系统
### 功能目标
为玩家提供稳定刷怪对象和成长压测对象。

### 首版内容
- [ ] 普通近战怪
- [ ] 普通远程怪
- [ ] 高速怪
- [ ] 厚血怪
- [ ] 精英怪 2 种
- [ ] Boss 2-3 个

### 验收标准
- 怪物区分度清晰
- 不需要复杂 AI
- 能稳定配合成长节奏提供压力

---

## 3.6 爬塔系统
### 功能目标
提供明确推进目标和无限延展空间。

### 任务
- [ ] 层数计数器
- [ ] 每层结束判定
- [ ] 每 5 层 Boss 判定
- [ ] 层数难度递增
- [ ] 失败后记录最高层

### 验收标准
- 用户知道自己在第几层
- 能明确感受到越往后越难
- 会产生“再高一层”的欲望

## 四、广告与商业化拆解

## 4.1 MVP 广告位
### 激励广告

- [ ] 局结束后低频触发

## 五、数值节奏拆解

## 5.1 前 60 秒
### 目标
- 快速刷怪
- 快速升级
- 迅速进入爽感状态

### 开发要求
- [ ] 怪物密度不要太低
- [ ] 升级门槛不要太高
- [ ] 首个强化影响要明显

## 5.2 前 3 分钟
### 目标
- 至少形成一条基础 Build
- 至少出现一次明显武器质变
- 至少打到一个小高潮

### 开发要求
- [ ] 武器升级路径清晰
- [ ] 中途不要过度空档
- [ ] 玩家不要长时间只是“慢慢磨怪”

## 5.3 死亡与重开
### 目标
- 死亡不让人烦
- 失败后迅速重来

### 开发要求
- [ ] Game Over 动画尽量短
- [ ] 再来一局按钮醒目
- [ ] 流程不要层层确认

## 八、测试重点

### 重点测试问题
1. 玩家是否能快速理解玩法？
2. 武器进化是否足够爽？
3. 三选一是否总有想点的东西？
4. 哪一层开始用户觉得无聊？
5. 死亡后愿不愿意再来一局？
6. 复活广告是否愿意点？

### 观察指标
- 首局时长
- 最高层数
- 重开次数
- 平均升级次数
- 广告触发率

---

## 九、开发原则总结

这款游戏开发时必须一直守住三条线：

### 1. 不要做复杂
不要让它变成“硬核横版动作游戏”。

### 2. 不要做空
不要让升级只是数值变大但没感觉。

### 3. 不要做慢
不要让用户前 2 分钟感受不到明显成长。

一句话总结：
> 这款产品的核心不是关卡设计，而是成长节奏设计；不是拼操作，而是拼武器进化和 Build 成型时的爽感。
